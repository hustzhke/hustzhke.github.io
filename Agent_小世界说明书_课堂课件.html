01 核心判断

这篇文章最重要的一句话

Agent 的门槛不是“能不能动手”，而是“知不知道自己正在什么样的工作世界里动手”。

常见误解

把 Agent 理解成“提示词 + 工具调用 + 工作流外壳”，等于把它看成更聪明的自动化脚本。

作者的反驳

工具调用只是行动通道。真正决定能否托付的，是它是否理解对象、流程、权限、例外与后果。

  • Prompt 可以让模型听懂命令，但不能自动补上业务边界。
  • 工具可以让模型执行动作，但不能保证它知道动作之后会发生什么。
  • 真正的差距，来自是否形成了具体场景里的世界模型。

02 OpenClaw Moment

为什么现在要重新谈 Agent？

1

能力成熟

底层模型、代码能力、电脑使用能力已经足够让人“真的试起来”。

2

门槛降低

更多人可以直接上手，而不是只停留在研究论文或实验室演示。

3

权限扩大

Agent 开始接触文件、软件、界面、流程与组织系统。

4

扩散加速

开源与模仿让“大家都能试”的速度大幅提升。

OpenClaw 的意义，不一定是发明了新算法，而是让 Agent 进入真实任务现场这件事，第一次大规模变得可见。

03 误解拆解

为什么“提示词 + 工具调用”不够？

工具调用解决什么

让模型能搜索、点按钮、读文件、改代码、发请求，本质上是在打开动作出口。

它没有解决什么

没有解决任务边界、组织责任、异常情况、错误后果、何时该停、何时该问人。

一个人会按按钮，不代表他理解这份工作；一个 Agent 会调用工具，也不代表它理解工具所在的世界。

  • 早期 AutoGPT 很火，但稳定做成的真实任务很少。
  • 原因不只是模型不够强，而是缺少工作世界里的判断标准。
  • “像 Agent 的外壳”不等于“可交付的 Agent”。
Slide 4 / 11 能执行，不等于能负责
04 小世界

Agent 真正要学的是什么？

对象

有哪些文件、角色、系统、状态、记录、输入与输出。

规则

流程怎么走，谁有权限，什么情况必须停下来，什么不能越权。

后果

一个动作会影响谁，错了会损失什么，出了问题先救哪一块。

“小世界”不是背景材料，而是专家能力本身。一个岗位、一个行业、一个组织流程，都可能是一个小世界。

  • 新手和专家都能点同一个按钮，但脑子里的后果模型完全不同。
  • 从实习生到老师傅，本质上就是逐步学会这个小世界。
  • Agent 要从演示变成生产力，也必须完成这一步。
Slide 5 / 11 小世界 = 对象 + 规则 + 后果
05 世界模型

为什么专家更可靠，而 Agent 常常卡在 60%-70%？

会操作

知道怎么点、怎么调、怎么执行。

懂流程

知道真实任务不是按钮串联，而是有边界。

懂权限

知道哪些动作必须审批，哪些不能越级。

懂例外

知道标准流程何时失效，何时该人工接管。

懂后果

知道哪类错误代价大，先防什么风险。

可托付

稳定交付，而不是偶尔惊艳、偶尔翻车。

最麻烦的不是 Agent 完全不会，而是“看起来很聪明，但关键 30%-40% 的地方常常摔跤”。

Slide 6 / 11 可靠性来自经验压缩，而不是每次临时推理
06 软件重估

软件为什么会从“工具”走向“劳动交付”？

过去的软件

  • 卖按钮、字段、视图、流程。
  • 用户自己学会使用，再把事情做完。
  • 价值在“能不能用”。

Agent 化的软件

  • 用户要的不是按钮，而是事情被完成。
  • 例如：线索被筛过、合同被初审、代码被改到可合并。
  • 价值转向“能不能稳定交付”。

当软件开始交付劳动，知识工作的很多部分就会被重新定价。真正值得担心的，不是科幻风险，而是工作替代与价值分配。

Slide 7 / 11 从 SaaS 到 “结果即服务”
07 课堂重点

比 Prompt 更重要的交付物：小世界说明书

1. 对象有哪些实体、文件、角色、系统、状态？
2. 流程一件事从开始到完成的真实路径是什么？
3. 权限谁能决定？谁能审批？哪些动作不能随便做？
4. 例外哪些情况不能套标准流程？
5. 标准什么叫“做对了”？谁说了算？
6. 失败过去最常见的错误是什么？后果是什么？
7. 风险哪些任务必须人审？哪些信息不能交给 Agent？
8. 反馈做完以后靠什么持续校正和学习？

Prompt 是命令，小世界说明书是上下文。没有上下文，再漂亮的命令也像在陌生办公室里大声喊话。

08 应用判断

哪些任务适合自动化，哪些值得做成专门化 Agent？

适合脚本

步骤清晰、风险低、例外少。重点是快，不一定需要“理解世界”。

适合人机协作 Agent

需要调用工具，但失败后果有限。让 Agent 先做执行，人类抽查与收尾。

适合专门化 Agent

权限复杂、隐性规则多、责任清晰、能沉淀数据与专家反馈，值得长期训练。

不要一上来就问“提示词怎么写”，先判断这项工作到底是脚本问题、协作问题，还是世界模型问题。

10 收束

一句话总结

未来最会用 Agent 的人，不一定是最会写 Prompt 的人，而是最能把自己的工作世界结构化、边界化、可校正化的人。

  • OpenClaw Moment 让 Agent 进入真实任务现场这件事变得可见。
  • Agent 的核心挑战从“动作能力”转向“世界理解”。
  • 谁能把小世界说清楚，谁就更有机会把经验沉淀成新的生产力。
